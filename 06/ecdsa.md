In this homework I implemented an ECDSA signing ad verification tool.

To sign some data, we need a hash (h) of the data and an elliptic curve private key (d), then:
1. Generate a random nonce k (must be kept secret and random) in the range [1, n-1], where n is the total number of points on the curve
2. Then calculate the random point R = k x G, and set r to the x coordinate of R
3. Then calculate the modular inverse k⁻¹, this can be done using the extended euclidian algorithm
4. Calculate s = k⁻¹ * (h + r * d) mod n
5. Signature results in r, s

For a 256bit curve r and s will be 32 byte values
Bit length of h must not exceed the bit-length of n
This method is probabilistic, because a different nonce is chosen every time

Public key is generated by multiplying the generator point G (base point, agreed common number in Diffie-Hellmann) with the private key integer d

For verification we need the hash (h), signature r, s, public key Q, then:
1. Calculate R' = (h * s⁻¹) x G + (r * s⁻¹) x Q
2. Verification is successful if the x-coordinate of R' = r mod n
